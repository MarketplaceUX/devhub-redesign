Design Story: Manage app versions


See the prototype

Thumbnail number 1 full-size

This design story covers

  1. Publishing an existing submission that's been saved but not yet submitted for review
  2. Resubmitting a version that's been reviewed as either "rejected" or "more information needed"
  3. Publishing a version that's been reviewed as "approved"
  4. Adding a new version of the manifest or packaged .zip file

User story 1: I want to continue an existing submission thatâ€™s been saved but not yet submitted for review

  1. Start on this page
  2. We have an app (version 1.1.4) that we haven't submitted for review yet. Click Submit.
  3. The app moves to the Submitted bucket.
  4. Let's find out what we submitted. Mouse over the app version area to maximize it. If a reviewer hasn't come back with a reply yet, you can't change the manifest or the release note.

User story 2: I want to resubmit a version of my app and provide an explanation, if the review team comes back saying "rejected" or "more information needed"

  1. Start on this page and go to the next instruction. Alternatively, after following the direction on Story 1, click Test: change to rejected. This button simulates what would happen if the app gets rejected.
  2. Our app comes back rejected. Now we can Update Manifest URL and tweak its Release Note.
  3. Before you resubmit, you may want to fill out the Reply text box for the reviewer. Then click Resubmit.
  4. The app's been resubmitted. Remember: you can always maximize or minimize the version area.

User story 3: I want to publish a version of the app that has been reviewed as "approved"

  1. Start on this page and go to the next instruction. Alternatively, after following the direction on Story 1, click Test: change to approved. This button simulates what would happen if the app gets approved.
  2. Our app is now approved. You can say some nice words to our reviewer. Release Note is locked because the app's been approved.
  3. Ready to publish the new version? Click Publish. Note how this item now moves under the Currently Active bucket.

User story 4: I want to add a new version of the manifest or packaged file, and submit it for review

  1. Start on this page and go to the next instruction. Alternatively, after following the direction on Story 3, this page will automatically show up.
  2. We want to publish a new version of the same app. Click the Add a new version button.
  3. Copy and paste your manifest URL, then click Validate. If your app is a packaged app, then this field will tell you to upload a .zip file rather than copy-paste a URL.
  4. After validating the file, note how the draft version now has a new version number and icon. This is done for demonstration purpose, to distinguish it from the old version. We are getting the new version number and icon from the manifest or the package .zip file.
  5. Give it a spiffy Release Note. Then when you're ready to submit this new version, click Submit.
