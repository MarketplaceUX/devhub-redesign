Information Architecture

Sitemap

Top-level pages

  1. My Submissions - My Submissions is a page listing the user's apps. If no apps, just have the CTA to "Submit a New App." Could have a dropdown in topnav listing the user's apps for quick reference, but this isn't scalable...
  2. My Payment Accounts - My Payment Accounts is a page listing the user's payment accounts. Could have a dropdown in topnav listing the user's accounts for quick reference, but this isn't scalable... If no payment accounts, explain what they are and when you need them..
  3. Tools - There is no Tools "page." Just a dropdown in topnav listing API, Test Validation, Test Installation + Receipt Handling, and Test In-app Payments.
  4. Support + Documentation - There is no "help" page. Just click-through link from topnav that goes to MDN.
  5. My Account - Sign Out is a link in the My Account dropdown menu.

Section: My Payment Account

  1. Edit a Payment Account - edit a single account. Includes "delete" action.

Section: My Account

  1. What's in this section?

Section: My Submissions

  1. Add/Edit App - This is the same series of pages, though some are not available in edit mode.
    1. Choose Screen Size - Select from mobile, tablet, desktop, or all. Choice is required before proceeding.
    2. Choose Platform - FFOS is required, so other platforms are in addition to FFOS. Show that FFOS is the main platform and allow user to add (checkbox) others. Android is the only other platform at this time. In prototype, show a couple other GenericPlatforms just to show that this is the future plan. This step is required, but user is not required to choose additional platforms. After this user goes to Upload Manifest page.
    3. Upload Manifest - only available when creating an app.
    4. Manifest, Version & Publishing - only available when editing an app. A page listing all versions for a single app. Allows user to promote a version and delete a version. The Add Version link does not lead into the Add/Edit App flow. Instead, it becomes a greatly simplified process that involves 1) uploading a new manifest or .zip file, 2) updating Release Note, and 3) replying to the reviewer if necessary. This process happens inline, on the same page.
    5. Add/Edit General Information - includes all the app details stuff.
    6. Add/Edit Categories
    7. Add/Edit Technical Information
    8. Add/Edit Team Members
    9. Add/Edit Content Rating
    10. Add/Edit Availability and Pricing
      1. Add/Edit App Availability
      2. Add/Edit App Pricing - includes In-app Purchases as part of this "page."
      3. Add/Edit Payment Account - Available only if user selects a pricing option requring a payment provider.
  2. User Feedback - if available. What does this look like?
  3. Analytics - if available. What does this look like?
